Youvi Technologies
Overview of technologies and Web APIs actually used in the Youvi platform

Contents

  • General Overview
  • Data Storage
  • Web APIs Used
  • Performance Optimization
  • Responsiveness

General Overview

Youvi is a local video hosting platform running entirely client-side. The system is built on modern web technologies and requires no backend for basic functionality (localhost server needed only for subtitle/multi-audio extraction via FFmpeg).

Key Feature: All data is stored locally on the user's device, ensuring complete content control and privacy.

Core Architecture Principles

  • Client-side first — all logic runs in the browser
  • Progressive Enhancement — basic functionality works everywhere, advanced features added when supported
  • Modularity — independent modules for different functions
  • Performance-oriented — optimized for fast performance with large collections (1000+ videos)

Data Storage

1. File System Access API

Foundation for working with local video files. Allows access to user folders and direct file manipulation.

  • Persistent handles — folder access preserved between sessions
  • Direct file reading — video access without copying to browser memory
  • Metadata — JSON files saved in .metadata folders
  • Previews — thumbnail generation and caching in .metadata folders
IMPORTANT: ALL video metadata (tags, comments, danmaku, descriptions, channel info, playlists) is stored in JSON files in .metadata folders next to videos, NOT in IndexedDB! File system is the source of truth.

2. IndexedDB

Client-side database used ONLY for autocomplete cache to avoid scanning folders every time.

  • tags — cached tag list for autocomplete
  • videos — cached video names for autocomplete
  • playlists — cached playlist names for autocomplete
  • channels — cached channel names for autocomplete
  • metadata — cache metadata (last update timestamp)

Source of truth: .metadata files. IndexedDB is only for UI speed optimization.

3. LocalStorage

Simple storage for UI settings and user preferences.

  • Theme selection (light/dark/skeuo)
  • Sidebar state (collapsed/expanded)
  • Player settings (volume, speed, quality)
  • Display preferences (grid/list)
  • Interface language (ru/en/uk)

Web APIs Used

1. Canvas API

Generating video previews and image processing.

  • Capturing frames from video for thumbnails
  • Image resizing and optimization
  • JPEG compression before saving (quality 0.85)
  • Generating timeline preview images

2. Broadcast Channel API

Real-time data synchronization between browser tabs.

  • Tag updates synchronized across tabs
  • Video metadata changes broadcasted
  • Change notifications
  • Theme switching synced across tabs

3. Intersection Observer API

Tracking element visibility for performance optimization.

  • Lazy loading of images (thumbnails load when visible)
  • Virtual scroll rendering in playlists
  • Mini-player video pause when outside viewport
  • Hover preview queue management

4. ResizeObserver API

Tracking element size changes for adaptive layouts.

  • Danmaku track recalculation on player resize
  • Subtitle scaling to player size
  • Document Picture-in-Picture window adaptation
  • Density graph updates

5. Picture-in-Picture API

Floating video window over other applications.

  • Document Picture-in-Picture — full mini-player outside browser
  • Continue playback when switching apps/tabs
  • Custom controls in PiP window
  • Danmaku support in PiP mode

6. FFmpeg.wasm (Web Workers)

Video processing for extracting subtitles and audio tracks via FFmpeg in Web Worker.

  • Extract subtitles from MKV (ASS/SRT/SSA formats)
  • Extract fonts from MKV attachments
  • Extract multiple audio tracks
  • Runs in separate worker, doesn't block UI
  • Requires localhost server (Python http.server)

Performance Optimization

1. List Virtualization

Rendering only visible elements to handle large collections.

  • VirtualPlaylistRenderer for tag lists
  • Tag cloud with virtual scrolling (500+ tags threshold)
  • Reduces DOM load by 10-100 times
  • Intersection Observer for visibility tracking

2. Lazy Loading

Loading resources only when needed.

  • Images load when appearing in viewport
  • Video previews generated on demand
  • Hover-preview with priority queue system
  • Intersection Observer for tracking

3. Multi-Level Caching

Multiple cache layers for fast access.

  • Memory cache — frequently used data in RAM
  • IndexedDB cache — autocomplete cache (tags, videos, channels, playlists)
  • Avatar cache — channel avatars cached in memory
  • .metadata cache — previews stored in file system

4. Image Optimization

Automatic processing to reduce file size.

  • JPEG compression with quality 0.85
  • Size limit (max 1920x1080)
  • Multiple thumbnail sizes generated
  • WebP for supporting browsers

5. Debouncing and Throttling

Limiting function execution frequency.

  • Search with 300ms delay
  • Scroll events throttled
  • Resize events debounced
  • Autosave with interval

Responsiveness and Accessibility

Responsive Breakpoints

  • 2000px+ — 6 columns, max content width
  • 1400-1999px — 6 columns
  • 1200-1399px — 5 columns
  • 1000-1199px — 4 columns
  • 768-999px — 3 columns, simplified header
  • 480-767px — 2 columns, mobile navigation
  • <480px — 2 columns, minimal UI

Adaptive Elements

  • Video grid dynamically changes column count
  • Sidebar collapses/hides on mobile
  • Header simplifies and collapses
  • Cards become larger on small screens

Theme System

Full support for three themes with smooth transitions.

  • CSS custom properties for colors
  • Classes: body.dark-theme and body.skeuo-theme
  • Choice saved in localStorage
  • Synced between tabs via BroadcastChannel
  • All components adapted to themes

Internationalization (i18n)

Support for three interface languages.

  • Russian (default)
  • English
  • Ukrainian
  • Dynamic switching without page reload
  • Language preference saved in localStorage

Platform for watching video content. Save, categorize and watch videos.
