Search Operators
Advanced search capabilities — boolean operators, exact search, wildcard and tag type search

Boolean operators

Boolean operators allow combining multiple search conditions for precise results.

AND

Find videos that contain ALL specified terms.

Writing variants: AND, &&, &

anime AND comedy
games && review
series & 2024

Result: Videos that have both first AND second tag.

OR

Find videos that contain AT LEAST ONE of specified terms.

Writing variants: OR, ||, |

anime OR manga
movie || series
comedy | drama

Result: Videos with first tag OR second tag (or both).

NOT

Exclude videos with specific term.

Writing variants: NOT, !, -

anime NOT horror
games ! sports
comedy - romance

Result: Videos with first tag but WITHOUT second.

Important: Operators are case-insensitive — you can write AND, and or And.

Combining operators

You can use multiple operators in one query:

category:Anime AND genre:Comedy NOT genre:Romance
games AND (review OR let's play)
anime AND 2024 NOT horror

Exact search

Use quotes to search for exact phrase. System will search for this exact word sequence.

Syntax

Enclose phrase in double quotes:

"new episode"
"anime 2024"
"game review"

Examples

  • "Cowboy Bebop" — will find only videos with exact phrase "Cowboy Bebop"
  • "Breaking Bad" — will find series, but not "Breaking" or "Bad" separately
  • "Windows 10" — will find exactly "Windows 10", not "Windows" or "10"

Combining with operators

Exact search can be combined with boolean operators:

"anime 2024" AND "new"
"Breaking Bad" OR "Better Call Saul"
"game review" NOT "spoilers"

Wildcard search

Use asterisk (*) as wildcard character for pattern search.

Syntax

Asterisk replaces any number of characters (including zero):

anime* — starts with "anime"
*series — ends with "series"
*2024* — contains "2024" anywhere

Usage examples

  • anime* — will find "anime", "animation", "animated"
  • *series* — will find "series", "miniseries", "web-series"
  • gam* — will find "game", "games", "gaming"
  • *2024* — will find everything with "2024" in title or tags

Combining

Wildcard can be combined with operators:

anime* AND comedy
gam* OR *play
*2024* NOT horror

Tag type search

Use prefixes to search by specific tag types. This allows searching only in specific category.

Syntax

Format: prefix:value

category:Anime
genre:Comedy
studio:Kyoto Animation

Available prefixes

  • channel:/ka: — search by channel
  • general:/gt: — general tags
  • category:/ct: — content category
  • genre:/ge: — genre
  • anime:/at: — anime title
  • serial:/st: — series title
  • movie:/mt: — movie title
  • animation:/nat: — animation title
  • character:/ch: — character
  • author:/au: or artist:/ar: — author/artist
  • studio:/st: — studio
  • year:/yr: — year
  • type:/tp: — type (TV, OVA, movie etc.)
  • rating:/ra: — rating

Multi-word values

For values with multiple words use quotes:

studio:"Kyoto Animation"
serial:"Breaking Bad"
anime:"Cowboy Bebop"

Query examples

category:Anime AND genre:Comedy
anime:Nyaruko OR anime:smth else idk, random title name here
year:2024 AND rating:PG-13
st:"Kyoto Animation" NOT tp:OVA
Alias support: System automatically resolves tag aliases. For example, studio:KyoAni will find "Kyoto Animation".

Query examples

Practical examples of using search operators for different tasks.

Simple queries

Find all anime comedies:
category:Anime AND genre:Comedy
Find game reviews or let's plays:
ct:Games AND (review OR let's play)
Find 2024 anime without horror:
category:Anime AND year:2024 NOT genre:Horror

Complex queries

Find Kyoto Animation anime, comedy or drama, but not OVA:
studio:"Kyoto Animation" AND (genre:Comedy OR genre:Drama) NOT type:OVA
Find Breaking Bad or Better Call Saul series:
serial:"Breaking Bad" OR serial:"Better Call Saul"
Find everything about Windows except Windows 11:
Windows* NOT "Windows 11"

Channel search

Find all channel videos:
channel:ChannelName
Find reviews from specific channel:
channel:ChannelName AND review
Tip: Start with simple queries and gradually make them more complex, adding operators as needed.

Technical information

Query parsing

Search system uses special parser for boolean query processing:

  • Query tokenization by spaces
  • Operator recognition (AND, OR, NOT)
  • Quote processing for exact search
  • Tag type prefix parsing (category:, genre: etc.)
  • AST (abstract syntax tree) generation

Operator precedence

Operators execute left-to-right with parentheses consideration:

  1. Parentheses () — highest precedence
  2. NOT ! — negation
  3. AND && — logical AND
  4. OR || — logical OR

Search algorithm

Each search type uses its own algorithm:

  • Regular search — fuzzy match with transliteration
  • Exact search — regex with word boundaries
  • Wildcard — regex with * replaced by .*
  • By tag type — exact match with alias resolution

Optimization

System uses several optimization techniques:

  • Transliteration cache — up to 500 variants
  • Search index — updates every 30 seconds
  • Variant limit — maximum 20 transliteration variants
  • Early stopping — search termination when limit reached

Developer functions

Available JavaScript functions:

  • parseBooleanQuery(query) — boolean query parsing
  • evaluateBooleanQuery(query, items, searchFn) — query execution
  • SearchTranslit.matches(text, query) — match check with transliteration
  • filterVideos(query) — video search
  • filterPlaylists(query) — playlist search
  • filterChannels(query) — channel search

System files

  • youvi/search/search-logic.js — main search logic
  • youvi_search.html — search page
  • youvi/autocomplete/ — autocomplete system
