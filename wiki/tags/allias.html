What are Aliases

Aliases are alternative names for tags. They allow you to enter tags in different ways, and the system will automatically find the correct variant.

Why are Aliases Needed?

  • Different Languages — can enter in Russian or English
  • Abbreviations — use short name variants
  • Avoid Duplicates — all variants lead to one tag
  • Convenience — enter as convenient, system will understand
Example: Tag Windows (gt) has aliases Windows, Винда, Win. You can enter any of these variants, and the system will automatically use Windows (gt).

Canonical Tag and Aliases

Canonical Tag — the main, correct tag name with type: Windows (gt)

Aliases — alternative names WITHOUT type: Windows, Винда, Win

How to Add an Alias

Step 1: Open Tags Page

Go to the All Tags page

Step 2: Find the Needed Tag

Find the tag you want to add an alias for. For example: Windows (gt)

Step 3: Open Editor

Hover over the tag and click on the pencil icon (✏️) to the right of the tag

Step 4: Add Aliases

In the opened window, enter aliases WITHOUT type:

  • Windows — short name without type
  • Винда — Russian variant
  • Win — abbreviation
Important: DO NOT add alias with type (for example, Windows (gt)). This will create a duplicate tag! Aliases are always WITHOUT type.

Step 5: Save

Click "Save" — aliases will immediately start working throughout the system

How it Works

Automatic Resolution

When you enter a tag, the system:

  1. Normalizes input (converts to lowercase, removes extra spaces)
  2. Searches in alias index
  3. Finds canonical tag
  4. Uses it instead of entered variant

Working Example

You enter: Винда, Win, windows

System resolves to: Windows (gt), Windows (gt), Windows (gt)

Result: video gets one tag Windows (gt) (without duplicates)

Case Insensitive

System automatically normalizes case:

  • Windows → windows
  • WINDOWS → windows
  • WiNdOwS → windows

All variants work the same!

Usage Examples

Operating Systems

Canonical: Windows (gt)

Aliases: Windows, Винда, Win

Result: can enter any variant, system uses Windows (gt)

Companies

Canonical: Microsoft (au)

Aliases: Microsoft, MS, Майкрософт

Result: abbreviations and translations work

Genres

Canonical: Комедия (ge)

Aliases: Комедия, Comedy

Result: Russian and English variants

Best Practices

What to Add in Aliases

  • Translations — Russian and English variants
  • Abbreviations — popular short forms
  • Alternative Spellings — different transliteration variants
  • Original Names — in original language (Japanese, Korean, etc.)

What NOT to Do

  • Don't add alias with type: Windows (gt)
  • Don't create too general aliases: Win for Windows 10 (gt)
  • Don't duplicate canonical tag in aliases

Tips

  • Add aliases gradually, as needed
  • Use clear and common variants
  • Check that alias doesn't conflict with other tags
  • Document non-standard aliases

Technical Information

Data Structure

Aliases are stored in tag database (.youvi/tag-database.json):

{
  "canonical": "Windows (gt)",
  "type": "gt",
  "aliases": ["Windows", "Винда", "Win"],
  "usageCount": 42
}

Alias Index

For fast lookup, an index is created:

{
  "windows (gt)": "windows (gt)",  // canonical → itself
  "windows": "windows (gt)",        // alias → canonical
  "винда": "windows (gt)",          // alias → canonical
  "win": "windows (gt)"             // alias → canonical
}

Normalization

All tags and aliases are normalized:

  • Convert to lowercase
  • Remove extra spaces
  • Trim beginning and end

API for Developers

Check alias resolution:

// Resolve alias to canonical tag
const tag = window.tagDatabaseManager.getTag("Винда");
console.log(tag.canonical); // "Windows (gt)"

// Get all tag aliases
const aliases = tag.aliases;
console.log(aliases); // ["Windows", "Винда", "Win"]

Performance

Alias resolution works in O(1) thanks to index. Lookup is instant even for thousands of tags.
